# Tableau to Power BI Validation Dashboard

A lightweight React application that validates Tableau `.twbx` migration outcomes and generates a PBIP-style Power BI project package from normalized abstract syntax tree (AST) metadata generated by the Altimate AI adapter contract.

## What it does

- Converts normalized Tableau and Power BI AST JSON into `public/validation_results.csv`
- Generates a PBIP-style Power BI project package from Tableau AST metadata
- Compares data sources, relationships, calculations, visuals, and filters
- Scores each validation item with status, severity, match score, and remediation effort
- Loads `public/validation_results.csv` into an interactive dashboard
- Provides filters by workbook, category, status, severity, and search term
- Shows KPI cards, charts, and a detailed remediation backlog

## Current upload flow

```text
Frontend upload .twbx
        ↓
POST /api/convert
        ↓
Express backend extracts Tableau AST
        ↓
Generated Power BI AST + PBIP-style project package
        ↓
public/validation_results.csv is refreshed
        ↓
Dashboard updates and exposes download links
```

The upload flow currently provides downloads for:

- `tableau_ast.json`
- generated `powerbi_ast.json`
- `validation_results.csv`
- `powerbi_project_package.zip`

The Power BI package contains a PBIP-style project. To create a true `.pbix`, unzip the package, open the `.pbip` file in Power BI Desktop, review the generated model/report, then save as `.pbix`.

## Altimate AI conversion target

The project now expects Altimate AI to convert native BI files into the normalized AST format documented in:

```text
AST_SCHEMA.md
```

The intended production flow is:

```text
Tableau .twbx file
        ↓
Altimate AI Tableau metadata reader
        ↓
Normalized Tableau AST JSON

Power BI .pbix file
        ↓
Altimate AI Power BI metadata reader
        ↓
Normalized Power BI AST JSON

Normalized AST pair
        ↓
scripts/compare-asts.mjs
        ↓
validation_results.csv
        ↓
Dashboard
```

## Extract Tableau AST from a `.twbx`

```bash
npm run extract:tableau -- "/path/to/workbook.twbx" sample-metadata/tableau_ast.json
```

For the Airbnb workbook used in this project:

```bash
npm run extract:tableau -- "/Users/abhimanyudwivedi/Downloads/Airbnb - Analytics Case Study.twbx" sample-metadata/tableau_ast.json
```

## Generate validation CSV from ASTs

Use the included sample AST files:

```bash
npm run validate:ast
```

Use custom AST files:

```bash
npm run validate:ast:custom -- path/to/tableau_ast.json path/to/powerbi_ast.json public/validation_results.csv
```

If no `.pbix` has been provided yet, you can compare against the placeholder target to show that Power BI mappings are missing:

```bash
npm run validate:ast:custom -- sample-metadata/tableau_ast.json sample-metadata/powerbi_missing_ast.json public/validation_results.csv
```

The converter compares AST nodes by stable node `id`. For example, Tableau and Power BI should both emit matching IDs such as:

```text
calc_net_revenue
visual_top_products
filter_order_date
```

## GitHub Actions PBIX worker

The app can trigger a Windows GitHub Actions worker after generating `powerbi_project_package.zip`.

Required environment variables for the API runtime:

```text
GITHUB_OWNER=<github org/user>
GITHUB_REPO=<repo name>
GITHUB_REF=main
GITHUB_TOKEN=<fine-grained PAT with Actions workflow dispatch permission>
PUBLIC_BASE_URL=<public URL where /downloads/... is reachable>
```

Optional callback support:

```text
PBIX_CALLBACK_URL=<public API callback URL>
PBIX_CALLBACK_TOKEN=<shared bearer token>
```

Workflow file:

```text
.github/workflows/package-pbix.yml
```

After upload, click **Trigger PBIX Worker** in the dashboard. The API dispatches the workflow with the generated PBIP package URL.

### Important packaging note

The workflow currently includes a placeholder packaging step. Replace that step with your licensed Windows packaging command, such as `pbi-tools compile`, Power BI Desktop automation, or Fabric export. Until that command is configured, the GitHub Actions worker proves orchestration and uploads a placeholder artifact rather than a native `.pbix`.

## Creating a true `.pbix` from the generated package manually

This Mac environment does not have Microsoft Power BI Desktop or a PBIX packaging engine installed. The app therefore generates the closest open, reviewable Power BI project handoff: a PBIP-style package.

After uploading a `.twbx`:

1. Download `powerbi_project_package.zip` from the dashboard.
2. Unzip it on a machine with Power BI Desktop installed.
3. Open the generated `.pbip` file.
4. Review the generated `model.bim`, DAX measures, placeholder source query, and report layout.
5. Replace placeholder data-source logic with your real Power BI source connection.
6. Save/export from Power BI Desktop as `.pbix`.
